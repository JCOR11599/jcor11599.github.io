Blackjack AI

This is a program does two things: evaluate a policy and try to create its own policy. So a policy is simply a rule that an AI follows. Here, our rule is "hit when under 17". Using Monte Carlo Tree Policy Evaluation we can see that this policy actually does pretty well, it wins about 40% of the time. We use a strategy called Q-learning to create our own strategy. This strategy works quite a bit better, but only leads us to just above 50% win rate. As a result, it's not very beneficial to play Blackjack, especially for a prolonged period of time, as it is unlikely you'll come out on top.

Sudoku Solver

Many small puzzle games we are NP-Complete, which basically means we don't have a really good algorithm to solve them. However, this doesn't mean we can't; people can solve sudoku puzzles really quickly. Here, I've written a sudoku solver using constraint solving and backtracking. There are x many permutations of a sudoku board, which are far too many solutions to search through to solve our problem. The idea of constraint solving is that we can use given information to drastically reduce the number of possible solutions we should search through. The idea of backtracking is that we recursively try or guesses, and we are able to know which decisions lead to wrong answers. There is an improvement to this, which is using a SAT solver. Sudoku is essentially a series of conditions that we need to satisfy to find an answer. When we do a series of movies that lead to a wrong answer, we can create a new condition that is essentially another constraint and urther reduce the number of poential solutions to try.
